# Mayor Contable (MT940)

## Objetivo

El objetivo del metodo es consultar el mayor contable del usuario en formato `json` o `swift` a traves del endpoint `GET /PosTrade/MT940`.

## Detalle

| Detalle | Valor |
|---|---|
| Nombre del metodo | `MT940` |
| Tipo de request | `GET` |
| Request HTTP | `{{url}}/PosTrade/MT940?currency={{currency}}&accountTypeID={{accountTypeID}}&dateFrom={{dateFrom}}&dateTo={{dateTo}}&alyc={{alyc}}&accountCodeID={{accountCodeID}}&format={{format}}&showAccountingEndOfDay={{showAccountingEndOfDay}}` |

## Roles que pueden utilizarlo

- `Multi Cel`
- `MC RM`
- `Administrador`
- `AN`
- `MC`
- `CEL`
- `xComitente`
- `Clearing`
- `CEL RM`

## Parametros

| Nombre | Tipo de dato | Descripcion | Es obligatorio |
|---|---|---|---|
| `currency` | `string` | Moneda a consultar. | Obligatorio |
| `accountTypeID` | `string` | Tipo de cuenta contable. Debe parsearse a `byte`. | Obligatorio |
| `dateFrom` | `string` | Fecha desde en formato `yyyyMMdd`. | Obligatorio |
| `dateTo` | `string` | Fecha hasta en formato `yyyyMMdd`. | Obligatorio |
| `alyc` | `string` | Codigo de ALyC. Si llega vacio, se normaliza a `0`. | No |
| `accountCodeID` | `string` | Cuenta especifica. Si llega vacio, se normaliza a `0`. | No |
| `format` | `string` | Formato de respuesta: `json` o `swift`. Default: `json`. | No |
| `showAccountingEndOfDay` | `boolean` | Si vale `true`, expone el cierre completo del dia. | No |

## Funcion de la BD utilizada

- Funcion principal: `[Contabilidad].[GetReporteMayorContable]`
- Funciones complementarias cuando no hay detalle:
  - `[Contabilidad].[GetReporteMayorContableCabecera]`
  - `[Contabilidad].[GetReporteMayorContableCabeceraSinMovimientos]`

## Filtros y parametros

### Parametros internos relevantes

| Nombre | Tipo de dato | Origen |
|---|---|---|
| `fechaDesde` | `date` | Proviene de `dateFrom`. |
| `fechaHasta` | `date` | Proviene de `dateTo`. |
| `monedaSigla` | `varchar` | Proviene de `currency`. |
| `tipoCuenta` | `tinyint` | Proviene de `accountTypeID`. |
| `miembroCompensadorCodigo` | `varchar` | Proviene de `alyc`. |
| `cuentaCodigo` | `varchar` | Proviene de `accountCodeID`. |
| `miembroCompensadorCodigoFiltroUsuario` | `varchar(max)` | Filtro del usuario autenticado. |
| `cuentaCompensadorCodigoFiltroUsuario` | `varchar(max)` | Filtro del usuario autenticado. |
| `mostrarContabilidadFinDelDia` | `bit` | Proviene de `showAccountingEndOfDay`. |

### Observaciones tecnicas

- Si `format=swift`, la BL transforma el objeto y renombra campos usando tags tipo `:20:`, `:25:`, `:60M:`, `:61:`, `:86:` y `:65M:`.
- Si no hay movimientos de detalle, la BL intenta reconstruir la cabecera contable igualmente.
- Si no hay datos ni en detalle ni en cabecera, la BL devuelve un objeto con saldos `0`.

## Validaciones/comentarios en la API

### Status code y responses posibles

| HTTP status | Cuando ocurre | Response posible |
|---|---|---|
| `200 OK` | Consulta exitosa | Mayor contable en `json` o `swift`. |
| `400 BadRequest` | Faltan `dateFrom` o `dateTo` | `"Invalid entry. Both date From and date To are required."` |
| `400 BadRequest` | `accountTypeID` no parsea a `byte` | Mensaje de parseo. |
| `400 BadRequest` | `format` invalido | `"Parameter 'Format' malformed."` |
| `400 BadRequest` | Error de parseo de fechas | Mensaje devuelto por `TryParseParameter`. |
