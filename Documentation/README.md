# Fuente documental

Esta carpeta concentra la documentacion funcional de trabajo.

## Subcarpetas

- `Endpoints/`: documentos canonicos por endpoint, agrupados por ruta API
- `Indexes/`: paginas contenedoras locales para metodos con mas de un verbo
- `Legacy/`: referencias historicas que no deben confundirse con endpoints activos
- `Context/`: documentos generales de apoyo, no usados como fuente principal cuando contradicen codigo o SQL

## Regla principal

Todo documento publicable a Confluence debe salir de `Endpoints/`.

Los documentos en `Indexes/`, `Legacy/` y `Context/` sirven como apoyo, navegacion local o trazabilidad, pero no forman parte del set publicable por defecto.
